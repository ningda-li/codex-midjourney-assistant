# Midjourney 能力地图

## 作用

本文件回答两个问题：

1. 这轮任务该优先用什么 Midjourney 能力
2. 这轮任务明确不该先用什么能力

能力选择必须先于 prompt 编写。  
所有正式任务都要先产出一个 `primary_strategy`，再决定辅助能力。

## 选择总原则

1. 文本 prompt 永远保留，参考能力只能增强，不能替代清晰文本
2. 一轮只解决一个主矛盾，避免同时让参考图承担“主体一致性 + 风格统一 + 构图指定”三种职责
3. 如果用户给的是抽象方向，先靠文本和风格能力定方向，不要急着锁死一致性能力
4. 如果用户要延续已有角色或物体，再上主体一致性能力
5. 如果用户真正缺的是风格基准，而不是构图细节，优先 `Style Reference / Moodboards / Personalization`
6. 如果能力兼容性和站点能力冲突，以 `references/version-routing.md` 与现场探针为准

## 能力卡片

### 1. 文本 Prompt

职责：

1. 说明主体、动作、场景、风格、构图、光照、用途和约束
2. 为所有参考能力提供语义锚点

优先使用：

1. 首轮探索
2. 用户没有现成参考图
3. 方向还不稳定

不要单独依赖：

1. 需要跨轮保持同一角色或物体
2. 用户已经给出明确视觉样张

### 2. Image Prompt

职责：

1. 借图像语义和构图关系给出整体方向
2. 为复杂场景、姿态、画面关系提供起点

优先使用：

1. 用户说“照这个画面关系来”
2. 需要借现成图找构图或题材语义

不要优先使用：

1. 只想复用风格
2. 只想延续同一角色

补充：

1. 图像提示更像“视觉起点”，不是精修工具
2. 需要更强控制时，再配合 `--iw`

### 3. Style Reference

职责：

1. 借参考图复用色彩、材质、笔触、灯光和整体视觉气质
2. 不负责复制人物、物体或构图

优先使用：

1. 用户先要“这个味道 / 这个画风 / 这个光感”
2. 批量图需要统一视觉调性

不要优先使用：

1. 用户想复用同一角色长相
2. 用户想锁定同一个产品外形

使用要点：

1. 文本 prompt 继续写“画什么”，不要写“把这张图改成”
2. 如果文字风格词和风格参考明显冲突，先删文字里多余风格词
3. 需要更强风格影响时，再加 `--sw`

### 4. Omni Reference

职责：

1. 让 Midjourney 复用人物、物体、载具或非人主体的形态特征
2. 解决“同一个对象跨图延续”的问题

优先使用：

1. 同一角色多轮延续
2. 指定某个产品、道具、载具或生物继续换场景
3. 用户说“就按这个主体继续”

不要优先使用：

1. 只是想借风格
2. 只是想借大构图

兼容提醒：

1. 官方文档说明它在 `V7` 可用，且会替代旧的 `Character Reference`
2. `--q 4`、`Fast Mode`、`Draft Mode`、`Conversational Mode` 与 Omni Reference 有兼容限制
3. 带 `--oref` / `--ow` 的结果再做编辑时，往往要先去掉引用
4. 默认 `--ow 100`，一般不要轻易高过 `400`

### 5. Character Reference

职责：

1. 作为旧链路兼容项处理角色一致性

使用规则：

1. 主站优先按 Omni Reference 思维处理
2. 只有历史任务或旧资产明确使用 `--cref` 时，才保留兼容解释

### 6. Personalization

职责：

1. 把用户长期审美偏好映射到默认出图倾向
2. 适合长期稳定的个人风格，而不是单轮硬约束

优先使用：

1. 用户有明确长期偏好，且已经解锁个人档案
2. 同一用户长期都要“更像他喜欢的东西”

不要优先使用：

1. 单轮任务要完全服从客户 brief
2. 用户当前项目风格和个人偏好冲突

使用要点：

1. `--p` 不是替代 prompt，而是给 prompt 加偏好背景
2. `--stylize` 越高，个性化影响越明显
3. `Like` Explore 图片会继续影响全局档案

### 7. Moodboards

职责：

1. 用一组图建立更宽范围的视觉方向
2. 适合项目级风格定调，而不是单图抄样

优先使用：

1. 品牌视觉方向图
2. 系列任务需要一套共同气质
3. 用户说“这不是一张图的问题，是整套感觉要统一”

不要优先使用：

1. 用户只给了一张很明确的风格图
2. 当前只是修一张结果的小问题

兼容提醒：

1. Moodboards 属于个性化体系
2. 官方文档说明它不能和 `--sv`、`--sw` 一起用

### 8. Describe

职责：

1. 把图变成一组可用文字
2. 用于“用户给图但说不清为什么喜欢”

优先使用：

1. 用户无法 verbalize 风格
2. 需要从参考图抽风格词和构图词

不要直接交付：

1. `Describe` 的原始输出不能直接当最终 prompt
2. 必须经过整理，只保留和当前任务有关的词

### 9. Prompt Shortener

职责：

1. 缩短过长 prompt
2. 帮助识别真正有效的描述核心

优先使用：

1. prompt 过长、堆词、多个风格互相打架
2. `V8.1 Alpha` 已明显触发长度压缩
3. 用户反馈“方向对，但画面越来越脏”

不要误用：

1. 还没想清楚任务时，不要先压短
2. 缺失关键约束时，不能靠压短解决

### 10. Raw Mode

职责：

1. 降低默认风格干预
2. 提高 prompt 服从度

优先使用：

1. 用户要更干净、更客观、更像设定图或产品图
2. 当前结果总被默认审美带偏
3. 文字渲染不稳时需要更字面

不要优先使用：

1. 首轮纯探索、需要模型多给一点惊喜
2. 用户本来就想要更强的艺术润色

### 11. Run as HD

职责：

1. 把已经接近可交付的结果提升到更高分辨率版本
2. 它是收尾动作，不是方向探索动作

优先使用：

1. 方向已定
2. 当前结果已经能交付，只差清晰度或成品感

不要提前做：

1. 方向没定时不要先烧成本
2. 用户反馈还集中在主体、构图、服装等根本问题时不要先 HD

### 12. Draft Mode

职责：

1. 用更低成本、更快速度做方向探索
2. 适合探索轮和 Style Creator 预览

优先使用：

1. 首轮探索
2. 提案轮
3. 需要快速试多个方向

不要优先使用：

1. 定稿轮
2. 已经进入高完成度收尾

### 13. Conversational Mode

职责：

1. 让用户用自然语言跟 Midjourney 共同推敲 prompt
2. 适合模糊创意先变成方向

优先使用：

1. 用户需求很抽象
2. 还在探索轮
3. 希望用对话或语音快速试方向

不要优先使用：

1. 要沉淀模板
2. 要交付稳定可复现 prompt
3. 要进入自动执行链

### 14. Style Explorer

职责：

1. 搜索、筛选和试用现成 `sref` 码
2. 快速找到接近的风格族群

优先使用：

1. 风格方向大致清楚，但还没有 style code
2. 需要先看社区现成 `sref` 候选

不要优先使用：

1. 当前核心问题是主体一致性
2. 用户明确要沉淀自己的长期风格码

### 15. Style Creator

职责：

1. 生成可复用的 `--sref` 风格码
2. 把风格探索沉淀成长期资产

优先使用：

1. 用户明确要“沉淀一套风格码”
2. 后面会反复复用同一种风格语言

不要优先使用：

1. 当前只是赶一轮正式出图
2. 风格方向还没定

### 16. Editor

职责：

1. 做局部重绘、扩图、Pan、Zoom Out、Remix
2. 解决“整轮方向对了，只差局部修正”的问题

优先使用：

1. 局部修
2. 扩画面
3. 同一张图上继续改

不要优先使用：

1. 整轮主体错了
2. 整轮风格错了

补充：

1. `V7 / V8.1 Alpha` 的编辑链当前都要带着 `V6.1` 兼容意识
2. 详见 `references/midjourney-editor-playbook.md`

### 17. Smart Select

职责：

1. 更干净地选中复杂区域
2. 为局部擦除或背景替换建立遮罩

优先使用：

1. 区域边界不规则
2. 背景和主体交错

不要优先使用：

1. 简单矩形区域
2. 整图重跑

### 18. Layers

职责：

1. 把额外图像资产叠入当前构图
2. 先摆位，再决定哪些区域交给模型重绘

优先使用：

1. 用户有自己的外部素材
2. 要先组合后重生

不要优先使用：

1. 只是想整体换材质
2. 只是想修一小块

### 19. Retexture

职责：

1. 保留现有结构和构图
2. 整体换材质、媒介或风格

优先使用：

1. 结构对了但风格错了
2. 想在同一图上整体重做视觉语言

不要优先使用：

1. 主体结构错误
2. 只是局部修复

### 20. Text Generation

职责：

1. 让图片里出现短词或短句
2. 更像标牌、标题、标签能力，不是专业排版能力

优先使用：

1. 海报标题
2. 招牌、霓虹字、包装短文案

不要高估：

1. 不要把它当长段落排版工具
2. 不要把“排版设计任务”直接等同成图片生成任务

### 21. Video

职责：

1. 把起始图做成短视频片段
2. 解决“关键帧已定，只差轻动态”这类任务

优先使用：

1. 起始图已定
2. 用户要短动态
3. 需要 loop 或 end-frame 过渡

不要优先使用：

1. 还没定静态关键帧
2. 用户其实要长叙事动画

补充：

1. 视频链有自己单独的参数边界
2. 详见 `references/midjourney-video-playbook.md`

### 22. Seed Lock

职责：

1. 给测试与 A/B 比较提供可复核起点
2. 帮助 `Run as HD` 等 seed-locked 动作理解同源关系

优先使用：

1. 做实验
2. 对比 prompt 小改动
3. 需要定位“到底是哪一条改动造成差异”

不要优先使用：

1. 想锁定人物身份
2. 想锁定长期风格
3. 在 `Turbo` 上承诺稳定复现

## 按任务类型的首选能力

1. 角色设计：`文本 Prompt` 为主，按需要叠加 `Style Reference`，延续角色时再上 `Omni Reference`
2. 角色设定图：`文本 Prompt + Raw Mode` 为主，需要统一脸和服装时再上 `Omni Reference`
3. 海报：`文本 Prompt + Style Reference` 为主，必要时配 `Image Prompt` 或 `Text Generation`
4. 场景概念图：`文本 Prompt` 为主，复杂画面可配 `Image Prompt`
5. 产品图：`文本 Prompt + Raw Mode` 为主，已有产品外观时优先 `Omni Reference`
6. 品牌视觉方向图：`Moodboards / Personalization + 文本 Prompt`，必要时先用 `Style Explorer`
7. 多轮延续任务：优先 `Omni Reference`，必要时叠加 `Style Reference`
8. 参考图驱动任务：先判断参考图到底承担“风格、主体还是构图”，再单独选能力
9. 局部修图或扩图任务：优先 `Editor`，按需要落到 `Smart Select / Layers / Retexture`
10. 视频任务：先定起始帧，再单独切到 `Video` 链
11. 风格系统沉淀任务：先 `Style Explorer`，明确后进入 `Style Creator`

## 输出约束

每轮至少要在 `solution_plan` 里写清楚：

1. `primary_strategy`
2. `recommended_capabilities`
3. `blocked_capabilities`
4. `reference_strategy`
5. `quality_target`

如果当前明确“不桥接任何原生能力”，也要显式写出来，而不是省略。
