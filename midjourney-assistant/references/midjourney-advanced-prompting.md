# Midjourney 进阶提示词与参数技巧

## 作用

这份文档只回答一件事：

当前版本的 Midjourney，哪些提示词技巧和参数技巧还值得用，哪些已经不能按旧习惯乱用。

## 先记版本边界

1. `V7` 主站是当前正式默认链
2. `V8.1 Alpha` 只在 `alpha.midjourney.com`
3. `Multi-Prompts (::)` 当前按 `V6 / Niji 6` 兼容链理解，不默认规划进 `V7 / V8.1`
4. `--no`、`--q`、`Omni Reference` 不要默认规划进 `V8.1 Alpha`
5. `--stop` 当前只按 `V6 及更早` 的 legacy 参数理解，不再默认进入 `V7 / V8.1` 规划

## 当前最稳的 prompt 结构

一条可执行 prompt，优先按这个顺序组织：

1. 主体
2. 动作或视角
3. 场景或背景
4. 风格与媒介
5. 构图
6. 光照
7. 用途或交付语境
8. 约束

不要一上来堆风格词、流程说明或平台操作说明。

## 当前仍然有效的官方提示词技巧

### 1. 用清晰视觉词，不用操作说明

当前仍然优先描述“你想看到什么”，而不是写：

1. 帮我改成
2. 把这张图处理成
3. 请模型做什么步骤

尤其在 `Style Reference`、`Image Prompt`、`Omni Reference` 场景里，文本更应该描述结果内容，而不是描述怎么改参考图。

### 2. 扩展词优先用“视觉轴”

官方当前仍然适合用这些轴扩 prompt：

1. 媒介
2. 时间时期
3. 情绪
4. 颜色
5. 环境

它们适合做探索轮的单轴变化，不适合一次全堆进去。

### 3. 长 prompt 不是默认更好

当前更稳的方式仍是：

1. 一条主目标
2. 一条主风格家族
3. 一到两条测试轴

如果已经超出平台长度限制，`V8.1 Alpha` 会触发新的 `Prompt Shortener`；这说明你更应该先压缩需求，而不是继续堆词。

### 4. 需要文字时，用双引号

从 `V6` 起，图片里的短文字可以靠双引号触发：

1. 用双引号，不用单引号
2. 只适合短词、短句、标牌、标题
3. 最好同时写出 `with the words`、`text`、`written`
4. 如果文字不稳，优先试 `Raw` 或降低 `Stylize`
5. 真正修小字错字，优先 `Editor / Vary Region`

### 5. 对话式输入只适合探索，不适合定稿交付

`Conversational Mode` 当前是创意沟通工具，不是稳定执行器：

1. 适合方向还模糊时先把想法说出来
2. 适合 `Draft Mode` 联动探索
3. 不适合需要可复核、可复用、可自动执行的正式 prompt 交付

如果任务要进入自动链或模板沉淀，最后仍要落回清洗后的英文 prompt。

## Image Prompt 技巧

1. 一张图 + 文本，适合“借画面关系，再补文本要求”
2. 多张图无文本，适合做视觉混合
3. 多张图 + 文本，适合复杂方向
4. `--iw` 当前按 `0–3` 理解
5. 最好先把参考图裁成接近最终画幅
6. 纯图片、不带文本的 prompt，不要再加 `--stylize` 或 `--weird`
7. `Image Prompt` 是借语义和构图，不是精确修改工具；精确修改优先 `Editor`

## Style Reference / Style Code 技巧

1. `Style Reference` 负责风格，不负责主体复制
2. `--sw` 默认 `100`，范围 `0–1000`
3. `V7` 图片型 `Style Reference` 当前默认 `--sv 6`
4. `--sv 4` 是旧版 `V7` 风格参考模型，只在明确需要复刻老效果时再用
5. `--sref random` 适合探索 style code，但不要直接拿去做最终稳定交付
6. style code 可以叠加，但叠加不等于融合；越叠越容易脏
7. `Moodboards` 不能和 `--sv`、`--sw` 一起规划

## Omni Reference 技巧

1. `Omni Reference` 只在 `V7` 主链使用
2. 只能用一张 Omni 图
3. 文本 prompt 仍然必须写清楚场景和动作
4. `--ow` 默认 `100`，范围 `1–1000`
5. 除非你刻意用很高的 `Stylize`，否则 `--ow` 最好低于 `400`
6. 如果同时用了高 `Stylize` 或 `--exp`，要意识到这些都会和 `--ow` 争夺影响力
7. 它更适合锁主体，不适合单独承担整套风格控制

## `--no` 的正确用法

### 什么时候值得用

1. 排除简单名词
2. 你真的需要硬排除某些元素

### 什么时候不要乱用

1. 多词短语
2. 容易被拆词误读的词组
3. 服装、裸露、人体这类敏感组合词
4. `V8.1 Alpha`

当前官方提醒要注意：

1. `--no modern clothing` 会被拆成分别读取
2. 这种写法可能导致误判
3. 这类需求更稳的做法，是直接正写“你要的服装类型”

## `--repeat` 的正确用法

1. 适合一次并行跑多组方向
2. 只在 `Fast / Turbo` 可用
3. 非常耗时，不适合盲目常开
4. 跑完后它不会保留在成品 prompt 里，重跑时要手动再加
5. `--sref random` 和 `--repeat` 一起用时，每组图会拿到不同 style code

## `--draft` 的正确用法

1. 只按 `V7` 规划
2. 适合探索轮、提案轮、Style Creator 预览
3. 当前官方口径是 `10x` 更快、`1/2` GPU 成本
4. 有满意 draft 后，再 `Enhance` 或正常质量重跑
5. 定稿轮不要继续停在 draft

## Seed 技巧

1. `Seed` 只适合实验和 A/B 测试
2. 不要把 `Seed` 当风格锁定器
3. 不要把 `Seed` 当主体一致性工具
4. 跨时段、跨设置、跨版本时不要承诺完全稳定
5. 用 `Turbo` 时不要指望 seed locking 可靠
6. `V8.1 Alpha` 当前可按“`99%` 相同”理解，但仍不是绝对复刻

## 当前不要再默认当通用技巧的东西

### 1. `Multi-Prompts (::)`

当前不要默认规划进 `V7 / V8.1` 正式主链。

### 2. `--stop`

当前只按 `V6 及更早` 的 legacy 参数理解，不再进入 `V7 / V8.1 / Niji 7` 的常规规划。

### 3. 先靠长 prompt 堆满一切

当前更稳的方式仍是：

1. 一条主目标
2. 一条主风格家族
3. 一到两条测试轴

### 4. 让参数替代理解

以下都不能替代需求理解：

1. `--chaos`
2. `--weird`
3. `--seed`
4. `--stylize`

参数只能微调，不负责救错题。
