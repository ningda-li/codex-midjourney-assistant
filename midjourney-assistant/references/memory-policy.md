# 记忆策略

## 读取时机

必须先完成 `brief` 编译，再做记忆检索。
不要在任务入口预加载全部历史记忆。

## 读取优先级

按这个顺序读取：
1. 用户画像
2. 项目记忆
3. 蒸馏经验
4. 页面变化记忆

## 检索规则

默认根据这些 `brief` 字段做相关性检索：
1. `goal`
2. `must_have`
3. `must_not_have`
4. `style_bias`
5. `project_id`

## 记忆消费层

检索出来的记忆不能只挂在任务对象上，必须经过消费层进入主链：
1. 用户画像里的风格偏好、内容偏好、禁忌，优先转成 prompt 生成约束
2. 项目记忆和蒸馏经验，优先转成延续性提示和高置信提示词线索
3. 页面变化记忆，不直接写进 prompt，优先转成提交和结果回读阶段的执行提醒
4. 暂时无法翻成英文的记忆项，不允许直接混进最终 prompt；要么转成警告，要么在后续术语层补录

## 输出

记忆检索脚本至少要输出 `memory_policy_snapshot`，包含：
1. 实际读取了哪些来源
2. 命中了哪些条目
3. 哪些来源没有命中

编排器还要额外生成 `memory_consumption_snapshot`，至少包含：
1. 被实际消费的用户画像偏好
2. 被实际消费的项目记忆条目
3. 被实际消费的蒸馏经验条目
4. 被保留为执行提醒的页面变化条目
5. 未消费原因或术语警告

## 写回原则

默认不写回本地记忆。

本地记忆只分四类对用户暴露，每类都必须由用户主动提及；用户只提到某一类时，只写那一类：
1. 模板：可复用的 prompt 结构、参数组合、风格方向、模板候选和 review queue。
2. 画像：用户长期偏好、禁忌、常用风格、质量倾向、画像 signal 和画像合并。
3. 日志：本次过程、结果摘要、运行日志、运行摘要、`checkpoint`、`active-task` 指针和项目上下文。
4. 经验：从本轮复盘里总结出的稳定规律、失败原因、下次改法和经验蒸馏。

典型触发说法：
1. `记录为模板` 只允许写模板。
2. `记录到画像` 只允许写画像。
3. `记录日志` 或 `排障` 只允许写日志。
4. `记录经验` 或 `复盘` 只允许写经验。

调用方显式传入 `--allow-memory-writeback` 时，视为四类全部允许。普通生图、普通手动 prompt 交付、普通自动执行后不能默认触发任何一类写回。

## 非持久产物

以下内容是本轮调用产物，不按本地记忆写回处理：
1. 用户或调用方显式传入的 `--task-file`
2. 用户或调用方显式传入的 `--output-file`
3. 自动执行后端运行所需的临时任务文件
4. 返回 JSON 里的 `runtime_receipts` 跳过说明

这些产物可以用于当前调用交付或调试，但不能替代持久记忆的显式 opt-in 规则。
