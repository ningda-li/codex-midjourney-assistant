# 记忆策略

## 读取时机

必须先完成 `brief` 编译，再做记忆检索。  
不要在任务入口预加载全部历史记忆。

## 读取优先级

按这个顺序读取：
1. 用户画像
2. 项目记忆
3. 蒸馏经验
4. 页面变化记忆

## 检索规则

默认根据这些 `brief` 字段做相关性检索：
1. `goal`
2. `must_have`
3. `must_not_have`
4. `style_bias`
5. `project_id`

## 记忆消费层

检索出来的记忆不能只挂在任务对象上，必须经过消费层进入主链：
1. 用户画像里的风格偏好、内容偏好、禁忌，优先转成 prompt 生成约束
2. 项目记忆和蒸馏经验，优先转成延续性提示和高置信提示词线索
3. 页面变化记忆，不直接写进 prompt，优先转成提交和结果回读阶段的执行提醒
4. 暂时无法翻成英文的记忆项，不允许直接混进最终 prompt；要么转成警告，要么在后续术语层补录

## 输出

记忆检索脚本至少要输出 `memory_policy_snapshot`，包含：
1. 实际读取了哪些来源
2. 命中了哪些条目
3. 哪些来源没有命中

编排器还要额外生成 `memory_consumption_snapshot`，至少包含：
1. 被实际消费的用户画像偏好
2. 被实际消费的项目记忆条目
3. 被实际消费的蒸馏经验条目
4. 被保留为执行提醒的页面变化条目
5. 未消费原因或术语警告

## 默认写回

`v0.3` 阶段 1 起，主链默认写回口径统一如下：
1. 进入正式编排就写 `checkpoint`
2. 只要存在 `project_id`，就写项目上下文
3. 自动模式真实执行完一轮后，默认写运行日志和运行摘要
4. 手动模式默认不写运行日志，只写 `checkpoint` 和项目上下文

`v0.3` 阶段 2 起，自动模式真实执行完一轮后继续追加：
1. `scripts/profile_signal_extract.py`
2. `scripts/profile_merge.py`
3. `scripts/experience_distill.py`

## 写回边界

默认不做：
1. skill 主体知识改写
2. 模板自动生成
3. 子 skill 自动创建

这些能力等 `v0.3` 后续阶段单独接入，不在阶段 1 主链里自动触发。
