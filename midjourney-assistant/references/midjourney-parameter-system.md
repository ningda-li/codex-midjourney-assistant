# Midjourney 参数系统

## 作用

本文件解决三件事：

1. 什么时间该加参数
2. 加哪类参数才真正有用
3. 哪些参数组合会把任务带偏或抬高成本

参数是控制层，不是主体内容层。  
没有清晰 `solution_plan` 时，不允许先堆参数。

## 总原则

1. 参数永远放在 prompt 文本末尾
2. 最终 `prompt_text` 只允许英文；中文只留在内部说明与诊断里
3. 探索轮少参数，收敛轮稳参数，成品轮锁关键参数
4. 一次只用真正解决当前问题的参数，不做“全家桶”
5. 如果结果偏差来自需求理解错误，不要用参数硬修

## 参数分组

### 1. 画幅与版式控制

常用参数：

1. `--ar`
2. `--tile`

适用：

1. 角色设定图、海报、封面、横版 KV、竖版提案
2. 需要先固定画面比例时

不该解决：

1. 主体错误
2. 风格错误

默认策略：

1. 人物全身常从 `--ar 2:3` 或 `--ar 3:4` 起
2. 海报和横幅按用途上宽比
3. 材质平铺或图案任务才上 `--tile`

### 2. 风格强度与服从度

常用参数：

1. `--stylize` / `--s`
2. `--raw`
3. `--sref`
4. `--sw`
5. `--p`

适用：

1. 需要控制模型默认审美干预
2. 要把风格参考拉强或收弱
3. 要接入长期偏好

判断规则：

1. 结果总是比 brief 更花、更润、更偏艺术时，优先考虑 `--raw`
2. 风格参考不够稳时，再提高 `--sw`
3. 需要长期偏好影响时，再启用 `--p`

风险：

1. `--stylize` 太高会把任务带回模型自带审美
2. `--raw` 太早开，会让探索轮失去意外性
3. Moodboards 不能和 `--sv`、`--sw` 一起规划

### 3. 探索与随机性

常用参数：

1. `--chaos` / `--c`
2. `--weird` / `--w`
3. `--draft`
4. `--repeat` / `--r`
5. `--exp`

适用：

1. 方向探索
2. 需要刻意拉开候选差异
3. 需要更快出一轮初稿

判断规则：

1. 探索轮才用高 `--chaos`
2. `--weird` 只给需要不寻常造型或视觉突破的任务
3. `--draft` 适合“先看方向，再决定要不要精修”
4. `--repeat` 只给明确要并行跑多组方向的任务
5. `--exp` 当前只当 `V7 / V8.1` 的实验性创意强度补充项，不作为默认必规划参数

风险：

1. 收敛轮还维持高随机性，会让结果持续跑偏
2. 商业交付轮乱用 `--weird`，容易破坏可落地性
3. `--repeat` 会放大错误 prompt 的成本
4. 高 `Stylize`、高 `--exp`、高 `--ow` 一起上，会互相争夺主导权

### 4. 一致性与参考强度

常用参数：

1. `--iw`
2. `--oref`
3. `--ow`
4. `--seed`
5. 兼容历史任务时的 `--cref` / `--cw`

适用：

1. 参考图对构图或语义影响不够时，用 `--iw`
2. 主体延续时，用 `--oref` 与 `--ow`
3. 需要做实验对比时，用 `--seed`

判断规则：

1. `--seed` 用于实验和复现，不是长期风格锁定工具
2. 主体一致性优先靠 `Omni Reference`，不是靠 seed
3. `--iw` 解决的是“参考图权重”，不是“风格强度”

风险：

1. 用 seed 试图保角色一致性，常常会失败
2. 把 `--iw` 拉太高，文本语义会被压住
3. 带 `--oref` 的结果继续编辑时，要先确认兼容性
4. `Omni Reference` 当前不兼容 `Fast Mode`、`Draft Mode`、`Conversational Mode`、`--q 4`

### 5. 成本、速度与质量

常用参数：

1. `--q`
2. `Run as HD`

判断规则：

1. 首轮探索不必先拉高质量
2. 只在接近成品时再上更高成本
3. `Run as HD` 只给已经接近可交付的结果

风险：

1. `--q 4` 会显著提高成本
2. 官方文档说明 `--q 4` 不兼容 `Omni Reference`
3. 把高成本动作放在错误方向上，只会更贵地失败
4. `V8.1 Alpha` 不要默认规划 `--q`

### 6. 模型与版本

常用参数：

1. `--v`
2. `--niji`

判断规则：

1. 默认先跟站点与项目环境走
2. 只有明确要切版本或风格引擎时才显式写

补充：

1. 当前官方默认版本是 `V7`
2. `alpha.midjourney.com` 的 `V8.1 Alpha` 仍属于快速变化区域，能力边界要以现场能力探针和版本文档为准
3. `V8.1 Alpha` 当前不要默认规划 `--q`、`--no`、`Omni Reference`
4. 动漫任务如果明确切 `Niji 7`，不要再把它和 `V8.1 Alpha` 混写

### 7. 排除、分支与风格版本

常用参数：

1. `--no`
2. `--sv`
3. `--repeat` / `--r`

判断规则：

1. `--no` 只适合排除简单名词，不要拿多词短语硬写
2. `--sv` 只在 `Style Reference` 明确要换版本时再显式写
3. `--repeat` 只给明确要并行跑多组方向的任务

风险：

1. `--no` 会逐词读取，容易误伤语义
2. `Moodboards` 不能和 `--sv`、`--sw` 一起规划
3. `--repeat` 只在 `Fast / Turbo` 有效，会快速放大错误 prompt 的成本

### 8. 视频专属参数

常用参数：

1. `--motion low`
2. `--motion high`
3. `--loop`
4. `--end`
5. `--bs`

判断规则：

1. 视频链只用视频专属参数
2. 普通图片链参数不要整组照搬进视频链
3. 只有在已经有稳定起始帧后再配视频参数

风险：

1. 视频当前只接受视频专属参数组合
2. 不要把 `sref / oref / image prompt` 当成视频链参数继续带入
3. `HD` 视频和 `Relax` 视频有套餐与模式限制

### 9. Legacy 参数

常用参数：

1. `--stop`

判断规则：

1. 当前只在明确使用 `V6 及更早` 时才解释
2. `V7 / V8.1 / Niji 7` 不再默认规划 `--stop`

风险：

1. 把 `--stop` 当成当前主链常规参数，会把任务拉回过时口径

## 分阶段参数策略

### 探索轮

1. 先定 `--ar`
2. 如需更多分歧，再加少量 `--chaos`
3. 必要时加 `--raw` 或低到中等 `--stylize`
4. 不要同时堆 `--chaos + --weird + 多参考 + 高质量`

### 收敛轮

1. 固定画幅
2. 降低随机性
3. 只保留真正帮助一致性的参考参数
4. 需要更稳角色时，优先 `--oref`

### 成品确认轮

1. 锁住画幅、主体策略、风格策略
2. 去掉多余探索参数
3. 只保留成品相关动作，如 `Run as HD`

## 高风险组合

1. `Moodboards + --sw`
2. `Moodboards + --sv`
3. `Omni Reference + --q 4`
4. `Omni Reference + Fast Mode`
5. `Omni Reference + Draft Mode`
6. `Omni Reference + Conversational Mode`
7. 方向未定时直接高 `--q`
8. 仍在探索阶段却同时用高 `--chaos`、高 `--weird`、多组参考和多次重复
9. 纯图片 prompt 再加 `--stylize` 或 `--weird`
10. 视频任务混入普通图片链参数
11. `V8.1 Alpha` 里继续规划 `--q`、`--no` 或 `Omni Reference`
12. `V7 / V8.1 / Niji 7` 常规任务里继续规划 `--stop`

## 输出约束

`parameter_strategy` 至少要回答：

1. 这轮要不要参数
2. 每个参数是为了解决什么问题
3. 有没有需要明确避开的组合

如果没有必要参数，允许输出空数组，但要说明“为什么不加更稳”。
