## 作用

这份文档只负责回答：

什么时候应该继续重跑整轮生图，什么时候应该直接进 `Editor` 做编辑。

## 当前编辑链基线

按 Midjourney 官方文档，截至 `2026-04-24`：

1. `Editor` 可在网站上统一做 `Remix / Erase(Vary Region) / Pan / Zoom Out`
2. `V8.1 Alpha` 图可以进入 Editor，但当前编辑功能仍按 `V6.1` 口径工作
3. `V7` 的 `Pan / Zoom Out` 也仍走 `V6.1`
4. `Editor` 分为从图库进入的轻量版和 `Edit` 页的完整版

所以编辑任务必须额外带着“这是编辑链，不是主生图链”的意识。

## 什么时候优先进 Editor

1. 只改局部
2. 只补边、扩图、腾空间
3. 主体和风格已经对了，只差某个区域
4. 需要同一张图上连续做多个局部动作
5. 结构对了，但想整体换材质或风格

## 什么时候不要急着进 Editor

1. 主体根本错了
2. 风格主方向根本不对
3. 还在探索轮
4. 需要的是整轮重写 prompt，而不是修某个局部

## Editor 里的几类动作

### 1. Move / Resize

适合：

1. 先改画幅
2. 给新元素腾位置
3. 先扩边，再补内容

官方当前支持：

1. 通过 Move / Resize 面板改画幅
2. 拖动画布边界扩空间
3. 通过自定义宽高比手动输入比例

### 2. Erase / Vary Region

适合：

1. 修手、脸、小物件
2. 局部替换
3. 给已有画面加新元素

使用规则：

1. 选区越大，变化越大
2. 选区越小，变化越细
3. 只描述选区里想改什么
4. 不要把整张图的新目标都写进局部编辑 prompt

### 3. Smart Select

适合：

1. 不规则物体边界
2. 复杂前景或背景
3. 需要先选中再擦除的区域

规则：

1. Smart Select 只是帮你更干净地建遮罩
2. 高亮后要真正执行 `Erase Selection` 或 `Erase Background`
3. 绿色高亮还在时，不代表已经提交编辑

### 4. Pan

适合：

1. 继续向左、右、上、下扩图
2. 保留原画中心内容，扩展外部空间

注意：

1. 当前 `V7` 里的 Pan 仍按 `V6.1` 编辑链理解
2. 如果目标是重新构图，不一定比整轮重跑更稳

### 5. Zoom Out

适合：

1. 拉远镜头
2. 给主体增加环境
3. 把“过近的图”拉回可用构图

注意：

1. `V7` 下也要按 `V6.1` 兼容链看
2. 如果只是想加少量空间，先比较 `Zoom Out` 和 `Move / Resize + Erase`
3. `Custom Zoom` 可在不真正拉远的情况下改宽高比

### 6. Remix

适合：

1. 编辑时顺便改文字引导
2. 同一张图做局部方向变化

规则：

1. Remix 只服务当前编辑目标
2. 不要在编辑阶段重新打开全新的大方向

### 7. Retexture

适合：

1. 构图和结构已经对了
2. 想整体换材质、媒介、光感或画风
3. 想保留原始结构但重做整体视觉外观

不适合：

1. 只改一个小区域
2. 主体结构本身错了

### 8. Layers

适合：

1. 需要把额外图像资产放进构图
2. 要把用户自己的图和 Midjourney 图组合
3. 要先摆位，再决定哪里交给模型重生

规则：

1. 哪一层被勾选，编辑工具就作用在哪一层
2. 图层提交后会被压平成单层
3. 如果想整张图统一成一个新风格，优先 `Retexture`

### 9. Suggest Prompt

适合：

1. 用户给了图但说不清楚怎么描述
2. 需要把上传图快速转成一条参考性 prompt

规则：

1. 它调用的是 `Describe` 思路
2. 只能当整理起点，不能原样无脑提交

## 外部图片编辑

当前官方支持：

1. 在 `Edit` 页面直接上传本地图片
2. 粘贴图片 URL

适合：

1. 用户上传自己的图再做编辑
2. 先扩画幅
3. 再用局部编辑补内容

## 当前最重要的兼容提醒

### 1. Omni Reference 结果进编辑链前先降引用

官方当前明确：

1. 带 `--oref` / `--ow` 的结果，不要直接拿去做 `Vary Region / Pan / Zoom Out`
2. 先进网站 Editor
3. 先去掉对应的 Omni 引用参数
4. 再继续编辑

### 2. 外部图层和编辑结果不会自动回主图库

官方当前说明：

1. 带外部上传图或多图层的编辑结果，不会自动出现在 `Create / Organize`
2. 想让它进入主图库，必须 `Upscale to Gallery`
3. 如果只是下载草稿，不代表主站已经收录这张结果

### 3. 不要把编辑链当“更稳的重跑器”

编辑链适合：

1. 局部修
2. 扩边
3. 保留大体方向
4. 在同一结构上做整体换风格

不适合：

1. 推翻主体
2. 推翻风格方向
3. 用来代替需求理解

## 当前实战判断顺序

1. 先问自己：是整轮错了，还是局部错了
2. 整轮错了 -> 回到主生图链
3. 局部错了 -> 进 Editor
4. 只是扩图或腾空间 -> 先 `Move / Resize` 或 `Pan / Zoom Out`
5. 只是局部替换 -> 先 `Erase / Vary Region`
6. 结构对、风格错 -> 先 `Retexture`
7. 要叠外部资产 -> 先 `Layers`
