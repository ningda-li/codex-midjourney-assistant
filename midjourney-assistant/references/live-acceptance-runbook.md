# 阶段5实机验收 Runbook

## 目标

这份 runbook 只负责真实 Midjourney 网页现场验收，不负责脚本级 smoke，也不替代内部回归。

进入本 runbook 前，必须先满足：

1. `python scripts/run_regression_suite.py` 全通过
2. 当前修改已经稳定落地
3. 本轮验收目标已明确是后台模式、前台模式、手动模式、反馈续跑或结果归因中的哪一项

## 通用记录口径

每个实机验收项都只记录三种结论：

1. 通过：真实页面行为和预期一致
2. 阻塞：被登录态、挑战页、站点异常、窗口状态或外部环境挡住
3. 失败：链路能跑，但行为不符合预期

每个结论至少附带：

1. 本轮模式
2. 本轮需求摘要
3. 最终 prompt 摘要
4. 页面结果摘要
5. 如果失败或阻塞，明确卡在哪一步

## 验收项 1：首次启动与首次引导

### 目标

确认首次启动时能正确进入引导，不把正式流程和首次测试说乱。

### 步骤

1. 清空本地首次启动状态
2. 启动 skill
3. 检查开场白是否包含：
   - 自我介绍
   - 自动模式与手动模式说明
   - 首次启动要做什么准备
   - 自动模式默认后台模式
4. 按引导完成首次测试
5. 再次启动 skill，确认不再重复首次引导，而是进入正常流程说明

### 通过标准

1. 首次启动说明完整
2. 首测结束后状态写回
3. 二次启动进入正常流程

## 验收项 2：后台自动模式

### 目标

确认 assistant 能在独立浏览器里完成真实提交、轮询、结果回读和最终总结。

### 步骤

1. 不要求用户先打开当前主浏览器里的 Midjourney
2. 直接发一个明确出图需求，并指定自动模式或默认自动模式
3. 检查 assistant 是否直接进入知识判断并开始后台执行
4. 等待后台独立浏览器完成提交和结果读取
5. 核对最终结果说明是否基于真实页面，而不是内部过程

### 通过标准

1. 能自行拉起独立浏览器并复用登录态
2. 能成功提交真实 prompt
3. 能正确轮询并判断完成
4. 能读回真实结果，不混淆历史任务
5. 用户可见输出不泄漏内部脚本和回归过程

## 验收项 3：前台自动模式

### 目标

确认 assistant 能复用用户当前 Midjourney 页面完成真实提交和结果回读。

### 步骤

1. 用户提前把 Midjourney 可操作页面打开在当前桌面
2. 以“前台模式”发起明确需求
3. 检查 assistant 是否不再切回后台模式，也不重新解释模式
4. 观察真实页面是否被正确复用
5. 核对最终结果和页面任务卡是否对应

### 通过标准

1. 使用的是当前页面，不是独立浏览器
2. 不改错窗口，不误碰无关页面
3. 提交 prompt 与结果卡对应正确
4. 最终总结基于真实结果，而不是脚本状态

## 验收项 4：手动模式

### 目标

确认 assistant 在不操作网页的前提下，仍然按同一知识主链给出专业交付。

### 步骤

1. 发一个明确需求，并指定手动模式
2. 检查 assistant 是否先理解任务，再输出手动交付
3. 检查交付是否至少包含：
   - brief 摘要
   - 最终英文 prompt
   - 参数或提交建议
   - 结果判断或反馈要求

### 通过标准

1. 不操作网页
2. 不退化成简单翻译
3. prompt 为英文唯一出口
4. 给出的建议与自动模式的知识判断一致

## 验收项 5：反馈续跑

### 目标

确认用户给修改意见后，assistant 能先诊断，再继续本轮，而不是把反馈当成新任务。

### 步骤

1. 先完成一轮手动或自动任务
2. 用户只给修改意见，例如“改成全身，保留风格，画面更干净”
3. 检查 assistant 是否识别为反馈续跑
4. 检查下一轮是否体现：
   - 保留项
   - 修改项
   - 本轮目标
   - 新一轮 prompt 或自动提交

### 通过标准

1. 不丢上一轮上下文
2. 不把反馈错误改写成全新需求
3. 能解释为什么这样收敛
4. 自动模式和手动模式都能接住反馈

## 验收项 6：结果归因与截图一致性

### 目标

确认 assistant 识别的是“哪张图、哪个任务卡、哪条 prompt”，而不是靠脚本状态瞎猜。

### 步骤

1. 让页面同时存在至少两轮任务卡
2. 继续做一轮修改
3. 检查 assistant 是否能明确说明当前判断对应的是哪一轮结果
4. 核对最终截图、任务卡和 prompt 是否一致

### 通过标准

1. 不把上轮图当成本轮图
2. 不把 prompt 文本误当成“已经出图”
3. 截图结论与页面真实结果一致

## 失败后的处理规则

1. 如果是脚本级、知识链级或内部输出泄漏问题，先回到内部回归修复
2. 如果是登录态、挑战页、网络异常或窗口焦点问题，按“阻塞”记录，不把它误判成知识链失败
3. 如果是结果归因错误、模式跑偏、prompt 不一致，按“失败”记录，并补回归用例防回退
