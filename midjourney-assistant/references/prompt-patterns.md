# Prompt 策略

## 原则

不要把 prompt 当成逐字翻译。  
先判断任务阶段，再决定 prompt 结构。

## 三阶段

### 1. 探索轮

适用：

1. 用户方向模糊
2. 需要快速试风格
3. 需要快速试构图

目标：

1. 快速得到方向
2. 不追求最终成品

### 2. 收敛轮

适用：

1. 已经有可用方向
2. 需要稳定主体、风格、用途一致性

### 3. 成品确认轮

适用：

1. 已经接近目标
2. 需要可交付结果

## 基础结构

v0.1 的 prompt 建议按这些槽位组织：

1. 主体
2. 场景
3. 风格
4. 构图
5. 光照与氛围
6. 用途导向
7. 约束

## 基础模板

```text
[主体], [场景], [风格], [构图], [光照与氛围], [用途导向], [必要约束]
```

## 生成时机

如果已经有稳定偏好或原生个性化能力可用，先判断是否桥接，再决定 prompt 是否需要更长。
