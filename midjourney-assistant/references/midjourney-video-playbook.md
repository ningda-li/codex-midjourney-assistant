## 作用

这份文档只负责视频链，不和普通图片主链混在一起。

## 当前官方基线

截至 `2026-04-24`，按 Midjourney 官方文档先按这套理解：

1. 视频是把一张起始图变成 `5` 秒动态片段
2. 输入核心是 `Starting Frame + 可选文本`
3. 视频参数只用视频专属那一组
4. 起始图原来带的图片参数会在进视频链时自动移除

## 什么时候值得用视频

1. 已经有一张定好的关键帧
2. 想给现有图片做短动态延展
3. 用户要的是气氛动效，不是长叙事动画

## 什么时候不要硬上视频

1. 起始图本身还没定
2. 用户要的是长剧情、多镜头剪辑
3. 用户其实只是想多出几张静态图

## 当前可用输入

### 1. Midjourney 自己生成的图

1. 可以直接从图库点 `Animate`
2. `Animate Manually` 可以先改文字再生成
3. `Auto` 直接按图生成
4. `Loop` 让起始帧和结束帧相同
5. 不论原图是哪个版本生成，只要在图库里都可以进入视频链

### 2. 用户自己的图

1. 可以上传到 `Starting Frame`
2. 也可以从上传库里选
3. 也可以在 Discord 用图片 URL 加 `--video`

## 当前参数边界

官方当前明确，视频生成只兼容这些参数：

1. `--motion low`
2. `--motion high`
3. `--raw`
4. `--loop`
5. `--end`
6. `--bs #`

不要把普通图片链里的参数直接带进去。

## 当前不要带进视频链的东西

视频链当前不要直接规划这些引用方式：

1. `Image Prompt`
2. `Style Reference`
3. `Omni Reference`
4. 普通图片链的旧参数尾巴

视频任务的核心参考是：

1. 起始帧
2. 可选结束帧
3. 可选短文本

## 运动模式

### 1. `motion low`

适合：

1. 轻微动作
2. 呼吸感
3. 更稳的镜头感

### 2. `motion high`

适合：

1. 更明显的运动
2. 动作夸张
3. 镜头感更强的短片段

规则：

1. 起始图不够稳时先不要上 `high`
2. 静态图越复杂，越要谨慎把运动幅度拉高

## 循环与结束帧

### 1. Loop

适合：

1. 呼吸感动画
2. 循环背景
3. 片段需要首尾衔接

### 2. End Frame

适合：

1. 明确从 `A` 过渡到 `B`
2. 想让视频有目标终点

规则：

1. `--loop` 表示首尾同图
2. `--end` 需要单独提供结束帧图像

## Batch Size 与成本

截至 `2026-04-24` 的官方口径：

1. 默认一次会生成 `4` 条视频
2. `--bs` 可设为 `1 / 2 / 4`
3. `SD` 的批量成本当前是：
   - `Batch 4 = 8 mins`
   - `Batch 2 = 4 mins`
   - `Batch 1 = 2 mins`
4. `HD` 的批量成本当前是：
   - `Batch 4 = 26 mins`
   - `Batch 2 = 13 mins`
   - `Batch 1 = 7 mins`

这组数值可能继续变，所以只按“当前官网口径”理解，不要写死成永久常量。

## 时长扩展

1. 初始视频是 `5` 秒
2. 每次 `Extend` 增加 `4` 秒
3. 最多扩展 `4` 次
4. 当前最大长度是 `21` 秒
5. 每次扩展成本和首轮视频生成相同

## 计划与模式提醒

当前官方口径：

1. 所有套餐都能在 `Fast` 生成视频
2. 只有 `Pro / Mega` 可在 `Relax` 跑视频，而且只限 `SD`
3. `HD` 视频只在 `Fast` 可用
4. 视频比普通图片更耗 GPU

## 当前实战技巧

1. 先把静态图定准，再做视频
2. 文本提示只补动作、节奏或氛围，不要把整张图重写一遍
3. 如果是轻微动态，优先 `motion low`
4. 如果要更明显的运动，再上 `motion high`
5. 想保留更受控的外观时，可以试 `--raw`
6. 如果只是想看动起来的感觉，先把 `--bs` 压到 `1` 或 `2`

## 当前常见误区

1. 以为视频能直接继承静态图的全部旧参数
2. 以为可以继续带 `sref / oref / image prompt`
3. 起始图还没定就开始做视频
4. 把视频链当成长片动画工具

## 当前判断顺序

1. 先确认有没有稳定起始帧
2. 再确认是普通动态还是 `loop / end-frame` 过渡
3. 再决定 `motion low / high`
4. 再决定 `--bs`
5. 最后才补少量文本
